# 実装（word_practice）

## 1. 目的
- 詳細設計に従って、MVPのタイピングソフト機能を実装する。
- 実装中に設計差分が出た場合は、先に設計ドキュメントを更新してから着手する。

## 2. 実装方針
- 技術前提:
  - Phoenix LiveView
  - SQLite3
  - ローカルLLM（分析・提案）
- 実装順序:
  1. データ層（スキーマ・マイグレーション）
  2. ドメインロジック（判定、SRS、集計）
  3. 画面実装（`/words`, `/practice`, `/review`, `/sessions/:id/result`）
  4. LLM分析・提案連携
  5. Docker起動手順の実装

## 3. 実装対象（MVP）
- ローマ字入力・漢字表示の練習セッション
- 段階ヒント（10秒/20秒）・30秒タイムアウト
- 回答履歴保存・単語/文字集計
- SRS更新・復習出題
- セッション結果表示
- 推奨単語表示（統計 + LLMハイブリッド）

## 4. 実装タスク
### 4.1 DB/スキーマ
- `words`
- `practice_sessions`
- `practice_answers`
- `word_stats`
- `char_stats`
- `vocabulary_fetch_logs`
- `analysis_runs`
- `word_recommendations`

### 4.2 ロジック
- 判定処理（`lemma_romaji`完全一致）
- ヒント制御・タイムアウト処理
- SRSレベル更新と`review_due_at`反映
- 単語統計・文字統計更新
- 推奨順位計算（rule 0.7 / llm 0.3）

### 4.3 UI
- 苦手単語管理画面
- 練習/復習画面
- 結果画面

## 5. 完了条件（工程6ゲート）
- 詳細設計で定義した機能が実装済み。
- 主要機能の動作確認が完了している。
- レビュー可能な差分として整理されている。
