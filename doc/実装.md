# 実装（word_practice）

## 1. 目的
- 詳細設計に従って、MVPのタイピングソフト機能を実装する。
- 実装中に設計差分が出た場合は、先に設計ドキュメントを更新してから着手する。

## 2. 実装方針
- 技術前提:
  - Phoenix LiveView
  - SQLite3
  - ローカルLLM（分析・提案）
- 実装順序:
  1. データ層（スキーマ・マイグレーション）
  2. ドメインロジック（判定、SRS、集計）
  3. 画面実装（`/`, `/settings`）
  4. LLM分析・提案連携
  5. Docker起動手順の実装

## 3. 実装対象（MVP）
- ローマ字入力・漢字表示の練習セッション
- スタート押下まで問題非表示、押下後に開始
- 入力文字の直接表示（テキストボックスなし）
- リアルタイム判定（判定ボタンなし）
- 段階ヒント（10秒/20秒）・30秒タイムアウト
- 回答履歴保存・単語/文字集計
- SRS相当重みの内部更新・自動ミックス出題
- 練習画面1ページでの結果表示
- 右カラムの苦手傾向表示（7件）
- 職業選択に応じたLLM問題生成と自動登録
