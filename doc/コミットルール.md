# コミットルール（word_practice）

## 1. 基本方針
- コミットメッセージは日本語で書く。
- 1コミット1目的にする。
- 件名だけで変更意図が分かるようにする。

## 2. 件名ルール
- 50文字程度まで。
- 「何を変えたか」を具体的に書く。
- 曖昧な語（`update` / `fix` / `WIP`）のみは使わない。

## 3. 本文ルール（必要なときのみ）
- 背景:
- 変更内容:
- 確認:

## 4. 例
- `要件定義の前提を練習ソフト向けに修正`
- `失語症調査結果をdoc配下に追加`
- `コミットルール文書を追加`

## 5. コミット前チェック
- `git status` で意図したファイルだけが含まれるか確認する。
- 変更とコミット件名が一致しているか確認する。
