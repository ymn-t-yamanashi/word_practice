# コードレビュー運用（word_practice）

## 1. 目的
- 変更の妥当性と品質を、マージ前に第三者視点で確認する。

## 2. 基本ルール
- 直接 `main` へコミットせず、Pull Request（PR）経由で反映する。
- PRには変更目的・変更内容・確認結果を記載する。
- レビューで指摘がある場合は修正して再レビューする。

## 3. レビュアー
- `CODEOWNERS` により、`@ymn-t-yamanashi` を既定レビュアーとする。

## 4. レビュー観点
- 要件整合: `doc/要件定義.md` と矛盾がないか
- 進め方整合: `doc/プロジェクト進め方.md` に沿っているか
- README整合: 変更がREADMEに反映されているか
- 差分妥当性: 変更範囲が目的に対して過不足ないか
- 再現性: 手順どおりに確認可能か

## 5. マージ条件
- PRテンプレートの必須項目が埋まっている
- レビュー観点の確認が完了している
- レビュー承認後にマージする

## 6. GitHub設定（手動）
GitHubのリポジトリ設定で以下を有効化する。
1. `Settings` → `Branches` → `Add rule`
2. Branch name pattern: `main`
3. `Require a pull request before merging` を有効化
4. `Require approvals` を有効化（1 approval）
5. `Require review from Code Owners` を有効化
6. 保存
