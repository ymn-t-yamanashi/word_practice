# word_practice

特定の単語が打てない課題を克服するための、タイピングソフト主軸の練習プロジェクトです。

## 現在のフェーズ
- 音声機能（録音・音声認識・音声解析）は未実装
- ドキュメントを基準に、ウォーターフォールで開発を進める
- Docker対応は実装仕様として要件定義/基本設計に反映済み

## 開発プロセス
- 開発モデル: ウォーターフォール
- 工程: 課題発見・方針決定 → 企画・構想 → プロジェクト立ち上げ → 要件定義 → 基本設計 → 詳細設計 → 実装 → テスト → リリース → 保守
- 詳細は `doc/プロジェクト進め方.md` を参照

## ドキュメント
- `doc/課題発見・方針決定.md`: 工程0の正本（課題、方針、Go/No-Go）
- `doc/企画・構想.md`: 企画・構想の正本（背景、目的、スコープ、KPI）
- `doc/プロジェクト立ち上げ.md`: 工程2の正本（体制、期間、運用ルール）
- `doc/基本設計.md`: 工程4の正本（画面、機能、データ構成）
- `doc/詳細設計.md`: 工程5の正本（入出力、判定、エラー処理）
- `doc/ER図.md`: MVPデータモデルのER図
- `doc/進行状況.csv`: 工程進捗の正本（現在工程の判断元）
- `doc/課題.md`: 課題メモ
- `doc/要件定義.md`: 要件定義（練習ソフト前提）
- `doc/失語症調査.md`: 失語症の調査結果
- `doc/コミットルール.md`: 日本語コミットルール
- `doc/プロジェクト進め方.md`: ウォーターフォールの進め方

## 運用ルール
- `word_practice` をリポジトリルートとして運用
- コミットメッセージは日本語で記述
- READMEに影響する変更（目的・構成・開発プロセス・主要ドキュメント）が発生した場合は、同じ変更で `README.md` も更新する
- 現在工程の確認は `bash scripts/progress_status.sh` を利用する
